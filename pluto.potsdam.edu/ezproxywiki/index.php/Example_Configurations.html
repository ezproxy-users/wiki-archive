Editing this page

Additions: Only add configurations you know work. Put them in alphabetical order.

Each database listing comprises:

  1. A level 4 header containing the database name (enclosed with 4 equal signs on each side)
  2. Optional comments
  3. A <pre> </pre> block containing the actual configuration
  4. A blank line.

See existing configurations if you are unsure.

Deletions: Only delete an entry if the site no longer exists. If the entry is wrong, and you don't know how to fix it, then add a note to the entry that says "This configuration is broken", don't just delete it.

Modifications: Only make changes that add functionality to an entry, or remove deprecated lines. If something doesn't matter (eg. redundant "D" lines, or "H" lines that are encompassed in a "D" line) then think carefully before making the change. If you don't know why a line is there, please leave it, as it may be relevant to specific configurations.

Unless you're about to edit numerous entries, please use the "edit" link to the far right of the entry in question, to be "sandboxed" into that one entry, and not risk mangling the rest of the page.

Please make judicious use of the "This is a minor edit" box, if your changes are trivial and do not impact functionality, or are just tidying or reordering.

Books 24x7

Updated configuration dated 2005-04-12 -- requires EZProxy 2.2a GA (2003-08-02) or later. Lukethelibrarian 11:59, 6 June 2006 (EDT) 

T Book24x7.com
U http://library.books24x7.com/library.asp?^B
Books24x7Site ABC123
TokenKey SomethingYouPickAndDontTellAnyone
TokenSignatureKey YouGetThisFromBooks24x7
DJ books24x7.com

where:

  • ABC123 is replaced by the site identifier issued to you by Books24x7.
  • SomethingYouPickAndDontTellAnyone is a random string that you pick that is used to encrypt the username of the person accessing EZproxy before sending it to Books24x7.
  • YouGetThisFromBooks24x7 is used to encrypt a combination of the IP address making the request and the encrypted username formed with TokenKey, or just the IP address if someone is accessing from within an ExcludeIP range.

Factiva

In May 2014 Factiva started causing security certificate warnings for some libraries using the config at OCLC's Factiva page dated 2012-12-04. A solution is to add two lines to the bottom of the stanza: 

Find document.getElementById('sendThemIn').action = "https://global.factiva.com.ezproxy.yourlib.org
Replace document.getElementById('sendThemIn').action = "https://global-factiva-com.ezproxy.yourlib.org

For some libraries it may be necessary to add the port number of your transformed Factiva hostname to the end of the second of those two lines, especially if you are not using the standard HTTPS port of 443. Eg, 

Replace document.getElementById('sendThemIn').action = "https://global-factiva-com.ezproxy.yourlib.org:2443

Gale InfoTrac

OCLC's Gale InfoTrac page has a stanza.

For many years, and still as of 2015-05-01, OCLC's stanza includes this line: 

DJ google.com

Some institutions would rather not proxy all of google.com, and it appears that this line is no longer required for Gale. This has been communicated to OCLC, but they have not removed it. It is suggested that this line be omitted. If you discover that it is required, please document which function in the Gale database requires it.

Email alerts

To have the EZproxy prefix added to URLs in email alerts:

  1. Log in to ProQuest's Local Administrator (LAD)
  2. Select "Account Preferences"
  3. Select "Search Interface Settings"
  4. Select the appropriate User Group
  5. Go down to General ProQuest Preferences - Where Link Prefix Default Value for Durable URLs
  6. Enter your EZproxy server address followed by "login?url=http://proquest.umi.com/pqdweb?". Eg, http://ezproxy.yourinstitution.edu/login?url=http://proquest.umi.com/pqdweb?

The Find and Replace lines serve to hide Wikipedia's editing tabs. If users edit Wikipedia articles, EZproxy will rewrite any proxied URLs in the article, including such URLs added by previous editors of the article. These links will be unusable by all the world except the editor's own institution. There are ways of editing Wikipedia that are not suppressed by this configuration, but this covers the most common.
