UsageLimit

From EZProxyWiki
Jump to: navigation, search

The UsageLimit directive is documented on OCLC's UsageLimit page, which should be referred to. Some clarification is useful though and there is additional documentation on OCLC's version changes page.

Clarifications

The "‑transfers" option would have been better named "‑requests" in the EZproxy software, as it applies to requests going out from the browser to the webserver, rather than the responses, ie, file transfers, coming back from the webserver. In practice, though, the difference is minor.

More significant is the use of the term "page requests" in the statement, "The ‑transfers specifies the number of page requests at which account access should be suspended." In fact, ‑transfers specifies the number of HTTP requests, that is, every request including those for images (such as gif, png and jpg files), script files (such as js files) and CSS files. On some websites, a single page view might generate dozens of HTTP requests and these all count towards the ‑transfers threshold.

The ‑transfers threshold works on the total number of HTTP requests, not just those that are recorded in ezproxy.log. In other words, if LogFilter directives are used to exclude certain types of files from ezproxy.log, a ‑transfers threshold may be reached while the log still contains fewer lines than the threshold.

When UsageLimits are applied in monitoring mode, that is, without the ‑enforce option, users are not denied access when they reach a threshold, but they do appear in the Admin module as "suspended".

Additional documentation

New UsageLimit options were introduced in EZproxy 3.6 in Q1 2006.

The new ‑IgnoreAutoLoginIP, ‑IgnoreRefererLogin and ‑IgnoreNormalLogin options exclude certain types of logins from participating in those limits. For example:

UsageLimit ‑enforce ‑interval=60 ‑expires=360 ‑MB=100 ‑IgnoreAutoLoginIP Global

enforces a limit of 100 MB transferred within a 60 minute window, with automatic expiration after 360 minutes, but ignores any access that occurs as a result of AutoLoginIP.

This information is not on OCLC's UsageLimit page, but on the Changes page.
