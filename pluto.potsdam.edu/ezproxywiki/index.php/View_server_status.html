View server status

From EZProxyWiki
Jump to: navigation, search

The Server Status screen of the Administration interface shows the version of EZproxy that is running, along with the last time the server was able to validate the license. This information is also logged in the messages.txt/user.msg file.

The Server Status page has six sections that are described below: Sessions, Databases, Login Ports, Hosts, Host Maintenance and Miscellaneous.

Sessions

Shows the open EZproxy sessions.

Databases

Shows all the databases that are defined in the config files.

Login Ports

Shows the login ports you are using. For ProxyByPort, the default is 2048, and for ProxyByHostname, may be one or both of 80 & 443.

Hosts

Shows the contents of the hosts file. You can trim this list by performing Host Maintenance.

Host Maintenance

Up to four options may appear:

  • No Action - does nothing
  • Remove x orphaned hosts that have !!! database indexes (requires EZProxy restart) - cleans up the host file, related to a proxied resource that is no longer in the config file, or perhaps changed hostnames.
  • Remove x hosts that have not been used in over 30 days (requires EZProxy restart) - cleans up the host file, related to a proxied resource that is no longer in the config file, or perhaps changed hostnames.
  • Reset accessed and referenced dates and counts of all hosts (no restart required)

Performing the host maintenance can be done at any time, even with the restart of the service. Removing hosts does not modify the config.txt (or ezproxy.cfg) file. It only removes a host from the hosts file - and next time the host is accessed, it will be added back into the hosts file.

Miscellaneous

The miscellaneous section shows the limit settings, and the peak values your server has experienced;

  • Peak Sessions active/limit: xxx/x,xxx
  • Peak concurrent transfers active/limit: xx/xxx
  • Peak virtual hosts/limit: x,xxx/x,xxx
