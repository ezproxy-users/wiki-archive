Database definition conflicts

From EZProxyWiki
Jump to: navigation, search

Starting in version 5.0, the EZproxy Admin page provides the ability to "Check for database definition conflicts". Don't be alarmed to discover "conflicts" – they do not necessarily mean you have a problem, and you may not have to do anything about them. A better term for this tool might be "database definition overlaps".

How it works

This tool looks at all the database definitions in the config file and compares them with each other. It flags pairs of definitions that have overlapping hosts or domains. Such overlaps might not cause a problem. You may deliberately have overlaps for a reason. On the other hand, the overlaps may be unintentional. Even then, they may be quite harmless. But sometimes definitions will not only overlap, they will actually conflict in a way that causes a problem with database access.

When there are overlaps, the first definition for a host determines how EZproxy handles the host. When trying to fix a problem with access to a host, tweaking the settings for it in the second definition will make no difference, because it is being overridden by the first definition (which you may be unaware exists).

When to check for database conflicts

1. Debugging problems

You SHOULD run this tool when you know you have a problem with access to a particular database and are not able to solve it easily. If the tool shows overlapping definitions for the host or domain, the cause of the problem may lie in the first of the overlapping definitions. You should fix the first definition. If the overlap is unintentional, you are probably best to consolidate the two definitions into one.

2. Housekeeping and preempting problems

You may run this tool if you wish to identify unintentional overlaps. It is probably a good idea to eliminate overlaps if you know there is no particular reason for them, even if they are harmless. This will simplify administration and preempt the possibility of the overlap becoming a problem in future.
