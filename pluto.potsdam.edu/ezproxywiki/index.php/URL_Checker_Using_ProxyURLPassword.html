URL Checker Using ProxyURLPassword

ProxyURLPassword is an EZProxy feature that allows one to pass EZProxy a structured XML document that contains a URL & a password and EZProxy will return a true or false value based on whether the URL has been setup for proxy in EZProxy. According to the release notes (http://oclc.org/support/services/ezproxy/documentation/changes.en.html) this was developed to support third-party applications, such as LibX. There is little official documentation on this feature, but excellent documentation on how to implement this feature using PHP (http://elibtronic.ca/content/20080821/checking-urls-connecting-ezproxy) and (http://elibtronic.ca/content/20101029/taking-care-campus-users-or-checking-urls-connecting-ezproxy-redux).

Below I offer a method to implement this feature using HTML and JavaScript. No additional resources are required, and this implementation should work with any standard release of EZProxy that supports ProxyURLPassword.

Submitted by Paul Butler, Library Technologies Support Analyst at Ball State University. http://cms.bsu.edu/academics/libraries/about/stafflisting/staffbydepartment/lits/butlerpaul

The Features

In addition to supporting the URL lookup, the code provides some additional features.

  • The page checks to confirm the user has input something into the input box, and if they have not, prompts them to do so.
  • The page checks to confirm if the user input begins with http or https, and if it does not, it prepends http:// to the URL before adding the proxy prefix.
  • The page checks to confirm if the URL contains any spaces, and removes any that are identified.
  • If the URL is setup for proxy the page returns a success message and offers the user a hyperlink with the URL and the prepended proxy string.
  • If the URL is NOT setup for proxy the page returns a failure message and offers the user a hyperlink with the EZProxy administrator's email address.

One issue with this implementation of ProxyURLPassword is that the password used to allow URL lookups is displayed in plain text in the source code of the page. This password is not used for any other feature or purpose, and only gives a user access to send a lookup request to HTTP://EZPROXY.YOURLIB.ORG/proxy_url. To mitigate risk you may place the page you provide to end-users in /loggedin, which will require the user to authenticate via EZProxy before accessing the page. More on this is provided below. The overall is low, but as with anything, caveat emptor!

The Setup

To implement this feature update config.txt with the following line: 
ProxyURLPassword somepassword

Replace somepassword with a new value and restart EZProxy. somepassword is shown in the source code of the HTML page, so it should be a random pattern of alphanumeric characters and NOT used for any other purpose. Once this is complete navigate to HTTP://EZPROXY.YOURLIB.ORG/proxy_url. If the page resolves correctly the feature is working.

The custom HTML page offered below should be located with your other EZProxy files. The exact location will determine who will have access to the page. The example URLs below assume the page is called URL_lookup.htm, you may name it whatever you want. (Thank you to Chris Zagar for explaining the different locations.)

  • Placing the HTML file in /public will allow anyone online to access the page. The resulting URL would be: HTTP://EZPROXY.YOURLIB.ORG/public/URL_lookup.htm
  • Placing the HTML file in /limited will require remote users to authenticate to access the page. The resulting URL would be: HTTP://EZPROXY.YOURLIB.ORG/limited/URL_lookup.htm
  • Placing the HTML file in /loggedin will force all users to authenticate before accessing the page. The resulting URL would be: HTTP://EZPROXY.YOURLIB.ORG/loggedin/URL_lookup.htm

The Code

What follows are the variables/values that need to be changed for the code to work properly. Searching the code for CHANGEME will help quickly locate these values.

proxy_string - The value for the variable proxy_string should contain your EZProxy prefix in the form of: HTTP://EZPROXY.YOURLIB.ORG/LOGIN?URL=

proxy_url_location - The value for the variable proxy_url_location should contain the location your proxy_url page in the form of: HTTP://EZPROXY.YOURLIB.ORG/proxy_url

YOUR_PASSWORD_HERE - Replace YOUR_PASSWORD_HERE with the same somepassword you used in the step above when you added ProxyURLPassword somepassword to config.txt.

EZPROXY-ADMIN@YOURLIB.ORG - EZPROXY-ADMIN@YOURLIB.ORG should be replaced with the email address of your EZProxy administrator, or whomever maintains EZProxy stanzas.

In the code block below is the code needed to create the HTML page. No CSS styling is provided. 

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
<html>
<head>
	<title>EZProxy URL Checker</title>

	<meta http-equiv="Content-Type" content="text/html;charset=utf-8" >
	
	<script type="text/javascript">
		function tranformInput() 
		{
			<!--formInput grabs the user's input for processing.-->
			var formInput = document.getElementById('userInput').value;	
			
			<!--A RegExp to remove space, tab, form feed, and line feed from the user input.-->
			formInput = formInput.replace(/\s+/g, '');	
			
			<!--This if-statement validates the form input, making sure the user has input something. In the input is blank a message is displayed prompting the user to add input.-->
			if (formInput==null || formInput=="") {
				alert("Please input a URL.");
				return false;
			}		
			else
			{				
				<!--CHANGEME: HTTP://EZPROXY.YOURLIB.ORG/LOGIN?URL= The value for the variable proxy_string should contain your EZProxy prefix. -->
				var proxy_string = "HTTP://EZPROXY.YOURLIB.ORG/LOGIN?URL=";
				
				<!--This variable and the if-statement evaluates the user input, formInput, to see if it starts with http or https, and if it does not prepends http:// to it. -->
				var urlPattern = /^((http|https):\/\/)/;
				if(!urlPattern.test(formInput)) {
					formInput = "http://" + formInput;
				}
				
				<!--The following lines of code submit the XML document, with the user's URL, to EZProxy to determine if the URL is proxied.-->
				
				<!--CHANGEME: HTTP://EZPROXY.YOURLIB.ORG/proxy_url - You must update the variable proxy_url_location with your institution's EZProxy address. The value must end with /proxy_url - which performs the URL lookup.-->
				var proxy_url_location = 'HTTP://EZPROXY.YOURLIB.ORG/proxy_url';
				
				var method = 'POST';
					
				<!--CHANGEME: YOUR_PASSWORD_HERE - Replace YOUR_PASSWORD_HERE below with the same somepassword you used when you added "ProxyURLPassword somepassword" to config.txt.-->
				var postData =
				  '<?xml version="1.0"?>\n'
				  + '  <proxy_url_request password="YOUR_PASSWORD_HERE">\n'
				  + '  <urls>\n'
				  + '  <url>'+formInput+'</url>\n'
				  + '  </urls>\n'
				  + '  </proxy_url_request>\n';
				var req = new XMLHttpRequest();
				req.open("POST", proxy_url_location, true);
				req.setRequestHeader('User-Agent', 'XMLHTTP/1.0');
				req.setRequestHeader('Content-type', 'application/x-www-form-urlencoded');
				req.onreadystatechange = function () 
				{
					if (req.readyState != 4) return;
					if (req.status != 200 && req.status != 304) {
						<!--Alert message shown to users if there is an error with the lookup. A good practice would be to include a contact info for the EZProxy administrator.-->
						alert('The proxy lookup failed, please contact your EZProxy administrator for assistance. HTTP error ' + req.status);
						return;
					}
					
					<!--This if-statement below evaluates whether the XML document returned from EZProxy /proxy_url contains true in it, which means the resource is setup for proxy. Otherwise, it contains false, and will be passed to the else statement. -->
					
					if(req.responseText.indexOf("true") > -1){
						<!-- Commented out, used for testing: alert("The test was TRUE!"); -->
						var formOutput = proxy_string.concat(formInput);
						document.getElementById('newURL').innerHTML = formOutput;
				
						var trueMessage = "The URL is setup for proxy, you may use the URL shown below.";
						document.getElementById('message').innerHTML = trueMessage;
				
						<!--The setAttribute sets the href for the link that is displayed. -->
						document.getElementById('link').setAttribute('href',formOutput);
					}
					else
					{
						<!--Commented out, used for testing: alert("The test was FALSE!"); -->
						var formOutput = "Please contact the EZProxy administrator before using this link with proxy.";
						document.getElementById('newURL').innerHTML = formOutput;
							
						var falseMessage = "The URL is not setup for proxy.";
						document.getElementById('message').innerHTML = falseMessage;
						
						<!--CHANGEME: Update the mailto address EZPROXY-ADMIN@YOURLIB.ORG with the email address for your EZProxy administrator, or whomever maintains EZProxy stanzas. -->
						var nullURL = "mailto:EZPROXY-ADMIN@YOURLIB.ORG?subject=EZProxy request for " + formInput;
						document.getElementById('link').setAttribute('href',nullURL);
					}
				}
				if (req.readyState == 4) return;
				req.send(postData);
				
			}
			
		}
	
	</script>
		
</head>

<body>

	<form name="theForm">

		<p>This page will check to see if a URL is setup and ready to use with EZProxy. If the URL is setup correctly the page will return the URL with the proxy prefix prepended to the URL. If it is not setup correctly for proxy a link is provided to email the EZProxy administrator to request the URL's addition to EZProxy.</p> 
	
		<input type='text' id='userInput' autofocus="autofocus"/><br>
		<input type='button' onclick='tranformInput()' value='Check the URL'/>
		
		<!--These spans display to the user all the work completed in the javascript.-->
		<p>
			<span id='message'></span><br>
			<a id="link" target="_blank"><span id='newURL'></span></a>
		</p> 
		
	</form>
	
</body>
</html>
