NeedHost alerts via Google Form

From EZProxyWiki
Jump to: navigation, search

At our library, we modified needhost.htm to provide users a way to report missing hosts to our staff with one click, right from that page.

To do so, we used the Forms feature of Google Documents. If you're not familiar with Google Forms, get a quick intro in this video from the Google Docs YouTube channel, and then refer to these help documents for details.

Once you're ready, then:

  1. Create a Google Form with two text fields: URL and email address.
    • I built a template for this form that you can copy and adapt: go to the template and click "Use this template" to begin.
  2. Save the form, and then click the link at the bottom of the form editing page to view the published form.
  3. In your browser, view the source of the published form. Find the following items:
    • <form> tag
      • note the URL that is its "action" parameter (will be something like https://spreadsheets.google.com/formResponse?formkey= followed by a long string of characters. This will be {form_action_url} in the code below
      • note the form's "id" parameter (probably "ss-form", which is what it will be in the code below)
    • <input> elements
      • note the name of each ("entry.0.single" and "entry.1.single" in the code below)
      • the id of each ("entry_0" and entry_1" in the code below)
  4. In your EZproxy docs directory, edit needhost.htm and insert something like this into the body of the page: 
<p>We're sorry.  There appears to be a problem with the library's setup of the material you are trying to access.</p>
<p>You can help us resolve this problem by clicking the "Submit" button below to let us know.</p>
<p>If you'd like us to email you when the problem is resolved, please enter your email address below before clicking "Submit".</p>
<form action="{form_action_url}" method="post" id="ss-form">
 <input type="hidden" name="entry.0.single" value="^V" id="entry_0">
 <input type="hidden" name="pageNumber" value="0"> 
 <input type="hidden" name="backupCache" value=""> 
 <label for="entry_1">Email:</label><input type="text" name="entry.1.single" value="" id="entry_1">
 <input value="Submit" type="submit" name="submit">
</form>
<p><strong>If the publication you're trying to access is a free publication,</strong>
<br/>you may also be able to access it directly from the publisher if you <a href="^V">click this link.</a></p>
<p>Thank you for your report.  We apologize for the inconvenience and will repair the problem as soon as possible.</p>
  1. Test by going to a Starting Point URL for a host that is not in config.txt (such as http://ezproxy.yourlib.org/login?url=http://www.npr.org)
  2. Submit the form, and then check the Google Spreadsheet that captures responses from the Google Form you created. You should see the timestamp of the submission, the URL of the problem page (i.e. http://www.npr.org in the above example), and the email address submitted.
  3. To receive email notification every time a new form submission occurs, open the Google Spreadsheet that captures your form's responses, pull down the "Tools" menu, and select "Notification rules...". Check the box "A user submits a form" to be notified when that occurs, then click "Save".
  4. Share the Spreadsheet with everyone on your library's EZproxy team so that they can also set up their own notifications and review the problem reports as needed.
  5. Leave notes to yourself or your team members about the status of problem reports by adding comments to the spreadsheet cells.
