Excellent start this. I'm just wondering though -- would it be more manageable to create a separate Wiki page for each site? I had originally started migrating the sections this way, all in the Category of "Configurations" ... It quickly got really stupid, adding dozens and dozens of 5-6 line "articles", so I deleted everything and started over with the present design. It does make forced alphabetization easier, though. :) --M@ 08:09, 21 April 2006 (EDT)

I thought the same as SteveThomas when I saw this -- obviously a lot of great and hard work put in here, but I think long-term it will get unwieldy to have these all on one page. The one-config-per-page works well for Chris in this part of the official support site, partly because it allows more space for explanation, variable expansion, etc. if necessary. If you decide to migrate this, please let me know and I'd be happy to help out with the moving process any way I can. And thanks again for all your work on this, M.
