Background

Single sign-on (SSO) functionality was added to EZproxy in version 4.0b (2006-08-18) allowing for the secure transfer of user authentication information to other systems. This article describes SSO implemented with PHP.

ezproxy.cfg Configuration

SSO -Secret=abcdefghijklmnopqrstuvw -URL=http://www.yourlib.org/sso.php abc

  • SSO doesn't require a database definition, so you do not need a "T" definition

-Secret

The SSO secret that appears in ezproxy.cfg must be 24 characters long, and it must match the first parameter of the call to EZproxySSO (in your PHP script).

-URL

In ezproxy.cfg, the URL should be to a script that invokes the EZproxySSO function.

The script that calls EZproxySSO should also include the URL of the EZproxy server.

abc

The "abc" that appears at the end of the SSO line in ezproxy.cfg and also appears in the URL to EZproxySSO needs to match.

The EZproxy server and PHP server should have their time set reasonably close. The script assumes the user will get from EZproxy to the PHP script within 60 seconds.

Further Information

An SSO definition does obey groups, so you could place a Group statement before the SSO line if you only want to have EZproxy pass select users back to your script.

This script does not allow the use of AutoLoginIP or referring URL since that makes no sense when you specifically want user information.

PHP Requirements

SSO functionality requires mycrypt to be installed on your PHP installation. To see if your PHP has this:

  1. Check using a phpinfo() call on any page and check the 'Configure Command' section for mcrypt
  2. Use a script such as the one described at http://us2.php.net/manual/en/function.mdecrypt-generic.php

Example PHP scripts are available at http://www.oclc.org/support/documentation/ezproxy/usr/sso/phpsso.tar
